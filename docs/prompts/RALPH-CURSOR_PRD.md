# Промпт для исполнения (рефакторинг PRD)

Работаем по Ральф-методологии. Ты — автономный агент, выполняющий задачи рефакторинга PRD из **docs/tasks/prd_refactor_tasks.json** (если файла нет — см. архив **docs/tasks/done/prd_refactor_tasks.json** после завершения очереди).

## Инициализация

1. Прочитай docs/new-agents.md (правила проекта)
   - Обрати внимание на секции **Build & Test** и **Known Gotchas** — это источник правды по подводным камням и конвенциям.
2. Прочитай docs/tasks/prd_refactor_tasks.json или docs/tasks/done/prd_refactor_tasks.json (список задач и agent_instructions)
3. Прочитай docs/PRD.md (целиком или разделы, указанные в выбранной задаче)
4. Выполни в **корне репозитория** (workspace root) команду: `git log --oneline -20` — чтобы увидеть последние коммиты; шаг не пропускать, выполнять через терминал.
5. Найди задачу для работы:
   - Статус: pending
   - Все dependencies имеют статус done
   - Наивысший приоритет (high > medium > low)

## Цикл работы над задачей

1. **Объяви задачу**: "Беру PRD-REFACTOR-XXX: <описание>"
2. **Реализуй** acceptance_criteria:
   - Открой `sections_to_review` и `target_section` в `docs/PRD.md`
   - Слей дублирующиеся требования в один канон Overeact Code
   - Обнови перекрёстные ссылки в PRD
   - Если правишь блоки про security/privacy/provider/permissions/terminal commands — формулируй **конкретные меры** (что именно внедрить), а не абстрактное “провести ревью/проверить”.
3. **Во время работы**:
   - Редактируй только docs/PRD.md; код не меняй
   - Если обнаружил новую gotcha/конвенцию, неочевидную из кода и доков — добавь **одну строку** в подходящую секцию docs/new-agents.md (не дублируй README и docstrings).
   - Сохраняй согласованность нумерации и ссылок внутри PRD
   - Не добавляй искусственный раздел 3.5: текущий PRD живёт в разделах 1–15, а backlog/open questions — в соответствующих существующих разделах.
4. **Проверь**: выполни все test_steps из задачи (ручная проверка PRD)
5. **Обнови docs/tasks/prd_refactor_tasks.json**: измени status на "done" *только после успешного прохождения test_steps*
6. **Запиши прогресс** в docs/tasks/progress.md (1–2 строки: что сделано)
7. **Сделай git commit** с осмысленным сообщением

## После завершения задачи

Напиши:

```text
PRD-REFACTOR-XXX выполнена.
Следующая pending-задача: PRD-REFACTOR-YYY (<краткое описание>).
```

## Правила

- Работай только над ОДНОЙ задачей за раз
- Не меняй и не удаляй задачи в prd_refactor_tasks.json — только status
- Если задача требует разделов или данных, которых нет в PRD — остановись, сообщи
- При противоречии между разделами (нумерация, ссылки) — исправь согласованность в рамках текущей задачи

## Когда все задачи "done"

- Перенеси docs/tasks/prd_refactor_tasks.json в docs/tasks/done. Для переноса используй команды командной строки.

Начинай!
